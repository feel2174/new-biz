# 네이버 검색광고용 작업형 랜딩 템플릿

네이버에서 `브랜드명`, `브랜드명 바로가기` 같은 순수 브랜드 키워드가 제한될 때는
페이지를 리뷰/정보 모음이 아니라 `브랜드명 + 작업/방법/문제 해결` 의도에 맞춘
작업형 안내 랜딩으로 작성한다.

## 운영 원칙

- CTA/AdSense 게이트는 네이버 `n_media`, `n_query`, `n_rank`, `n_ad_group`, `n_ad`, `n_keyword`, `n_keyword_id`, `n_campaign_type` 또는 `utm_source=naver&utm_medium=powerlink/cpc` 유입에서만 열린다.
- 구글은 `gclid`, `gclsrc`, `gad_source`, `wbraid`, `gbraid` 또는 `utm_source=google&utm_medium=cpc` 신호만 유료 유입으로 본다.
- 직접 접속, referrer 단독, `utm_medium=cpc` 단독, `NaPm` 단독, 카카오/다음 신호는 검수용 비노출 상태로 둔다.
- H1과 `primaryKeyword`에는 구매할 핵심 롱테일 키워드를 그대로 넣는다.
- `intent`는 방문자가 실제로 하려는 작업을 한 문장으로 쓴다.
- `summary`에는 페이지가 해결하는 작업을 2~4개로 나눈다.
- `quickLinks`는 유료 유입에서만 노출되는 상단 CTA 링크다. 직접 접속 검수 화면에서는 숨긴다.
- `cta`도 기존 정책대로 유료 유입에서만 강조 노출된다.
- `tasks`에는 앱 설치, 고객센터, 홈페이지 로그인, 청구, 조회처럼 작업 단위 섹션을 만든다.
- `relatedKeywords`는 순수 브랜드 키워드보다 `브랜드명 + 방법/설치/고객센터/조회/청구` 형태를 우선한다.
- 공식 주체가 아니면 `공식`, `혜택 보장`, `즉시 승인`, `최저가` 같은 표현을 피한다.
- 금융/보험/의료/대출은 자격, 심사, 비용, 보장 범위를 단정하지 않는다.
